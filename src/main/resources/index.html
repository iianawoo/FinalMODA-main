Design Patterns Used
All 7 patterns implemented in this project
Creational #1

Singleton

OrderManager и ProductCatalog — гарантируют один экземпляр на всё приложение.

Compile time: private конструктор запрещает new. Runtime: объект создаётся при первом вызове getInstance().

Creational #2

Builder

Order.Builder — собирает заказ шаг за шагом: имя, адрес, товары, доставка, упаковка.

Compile time: типы полей известны. Runtime: build() создаёт неизменяемый объект Order.

Structural #1

Decorator

PricingCalculator — оборачивает базовую цену слоями: доставка, экспресс, подарочная упаковка.

Runtime: цепочка декораторов формируется по опциям заказа.

Structural #2

Composite

CartItem — и одиночный товар (SingleItem), и набор (Bundle) реализуют один интерфейс. Корзина работает с ними одинаково.

Compile time: полиморфизм через интерфейс.

Structural #3

Facade

ShopFacade — один метод placeOrder() скрывает Builder, Decorator, Composite, Singleton. HTTP-обработчик не знает о деталях.

Compile time: минимум зависимостей у сервера.

Behavioral #1

Observer

OrderObserver — при событии заказа уведомляются ConsoleLogger, EmailNotifier, SmsNotifier.

Runtime: наблюдатели регистрируются при старте, уведомления идут автоматически.

Behavioral #2

Strategy

DiscountStrategy — WELCOME10, VIP20, SALE30 — разные алгоритмы скидки. Выбираются в runtime по промокоду без if/else в основном коде.
